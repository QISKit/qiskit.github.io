Provider for remote IBMQ backends with admin features.

This class is the entry point for handling backends from IBMQ, allowing using different accounts.

backends(name=None, filters=None, **kwargs)[source]¶

Return all backends accessible via IBMQ provider, subject to optional filtering.

Parameters:
  • name (str) – backend name to filter by
  • filters (callable) – more complex filters, such as lambda functions e.g. IBMQ.backends(filters=lambda b: b.congiguration[‘n_qubits’] > 5)
  • kwargs – simple filters specifying a true/false criteria in the backend configuration or backend status or provider credentials e.g. IBMQ.backends(n_qubits=5, operational=True, hub=’internal’)
Returns:

list of backends available that match the filter
Return type:

list[IBMQBackend]
Raises:

IBMQAccountError – if no account matched the filter.

Parameters:
  • token (str) – Quantum Experience or IBM Q API token.
  • url (str) – URL for Quantum Experience or IBM Q (for IBM Q, including the hub, group and project in the URL).
  • **kwargs (dict) –
    • proxies (dict): Proxy configuration for the API.
    • verify (bool): If False, ignores SSL certificates errors
load_accounts(**kwargs)[source]¶

Load IBMQ accounts found in the system into current session, subject to optional filtering.

Automatically load the accounts found in the system. This method looks for credentials in the following locations, in order, and returns as soon as credentials are found:

  1. in the Qconfig.py file in the current working directory.
  2. in the environment variables.
  3. in the qiskitrc configuration file
Raises:IBMQAccountError – if no credentials are found.
